# Motion And SFX

Use this file when a plan includes motion, click cues, transition, or SFX.

## Motion Rules

- Motion should point attention, reveal structure, or pace the edit.
- For screen recordings, prefer small focus, shimmer, cursor/click, path, marker, and label motion.
- For talking-head packaging, lower thirds, chapter cards, quote cards, app showcases, and transitions can be stronger.
- Do not hand-write arbitrary SVG/CSS/JS/GSAP for common assets. Use sourced assets, catalog elements, or CLI templates.
- GSAP and other runtimes may be used only when emitted by the CLI allowlist.

## SFX Rules

- SFX should sync to a visible or spoken event.
- Tie each SFX to a real UI action, spoken payoff, transition, or notification moment.
- Use click/pop sparingly and at low volume.
- SFX without a reason should be removed.
- SFX in screen recordings should support a UI action, not decorate every beat.
- Do not use SFX as filler for an empty asset plan; no SFX is valid when it better serves clarity or authenticity.

## Music

Music is not an SFX substitute. Use music only when the confirmed proposal needs packaging, retention, or mood. Speech-first videos should use low volume and ducking.

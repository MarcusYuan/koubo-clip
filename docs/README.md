# 文档

koubo-clip 文档用于把产品决策沉淀成稳定的实现计划。

从这里开始读：

- `../README-CN.md` - 用户安装、API/MCP 配置和使用方法。
- `requirements.md` - 初始规划讨论形成的产品需求。
- `architecture.md` - CLI 加 skills 的架构。
- `artifact-lifecycle.md` - artifact 权威性、fingerprint、依赖失效、恢复和宿主接入的目标合同。
- `artifact-lifecycle-implementation-plan.md` - 上述目标合同的阶段实施、回归矩阵和完成标准。
- `reference-learnings.md` - 从 Hermes、video-use、easy-video 和 OpenMontage 中借鉴什么。
- `release.md` - main、PR、测试版和正式版的发布流程。
- `internal-packaging.md` - 可选内部二进制 tarball 打包和安装说明。

Docs 面向用户、维护者和 agents。长期稳定的边界应放在 `rules/`。
